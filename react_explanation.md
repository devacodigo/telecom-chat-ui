## What is React and why we used it

React is a JavaScript library for building user interfaces. Instead of writing HTML directly, you write **components** — reusable pieces of UI that manage their own data.

**Why React over plain HTML:**
Plain HTML is static. If you want to update the page when something happens (user sends a message, agent replies), you'd have to manually find the element and update it. React does this automatically — you just update the data and React figures out what changed and updates the screen.

---

## What is JavaScript in this context

JavaScript is the programming language of the browser. Everything that happens in a webpage — button clicks, sending requests, updating content — is JavaScript.

React is written in JavaScript. When you write React, you're writing a special version called **JSX** (JavaScript + HTML mixed together).

### The component function

```javascript
export default function App() {
  // everything inside here
}
```

**What** — defines a React component called `App`.
**Why `export default`** — makes this component available to other files. `index.js` (auto-generated) imports `App` and renders it into the browser.

A React component is just a function that returns HTML-like code (JSX). React calls this function every time something changes and updates the screen with whatever it returns.

### useState — How React stores data

```javascript
const [messages, setMessages] = useState([
  {
    role: "agent",
    text: "Hello! I'm your Deutsche Telekom support assistant..."
  }
]);

const [input, setInput] = useState("");
const [loading, setLoading] = useState(false);
```

**What** — creates three pieces of state (data that can change).
**Why useState** — in React, you can't just use normal variables. If you do `let count = 0; count = 1`, React doesn't know anything changed and won't update the screen. `useState` tells React "watch this value — when it changes, re-render the component".

Each `useState` returns two things:
- The current value (`messages`, `input`, `loading`)
- A function to update it (`setMessages`, `setInput`, `setLoading`)

You always update state using the setter function, never directly:
```javascript
// Wrong — React won't notice
messages.push({...})


// Correct — React re-renders when setMessages is called
setMessages(prev => [...prev, newMessage])
```

**`prev => [...prev, newMessage]`** — this is an arrow function. `prev` is the current messages array. `...prev` spreads all existing messages, then we add `newMessage` at the end. This creates a new array instead of modifying the existing one — React requires this.

---

### useRef — Pointing at a real DOM element

```javascript
const messagesEndRef = useRef(null);
```

**What** — creates a reference to a real HTML element.
**Why** — we need to scroll to the bottom of the chat when new messages arrive. To scroll to something, you need a direct reference to it. `useRef` gives you that.

Later in the JSX:
```javascript
<div ref={messagesEndRef} />
```

This invisible div sits at the bottom of the chat. When we call `messagesEndRef.current.scrollIntoView()`, the browser scrolls to it.

// the broser goes to the last after this

### useEffect — Running code when something changes

```javascript
useEffect(() => {
  messagesEndRef.current?.scrollIntoView({ behavior: "smooth" });
}, [messages, loading]);
```

**What** — runs a function every time `messages` or `loading` changes.
**Why** — we want to auto-scroll to the bottom every time a new message appears. We can't do this inside `setMessages` because the DOM hasn't updated yet at that point. `useEffect` runs after React has updated the screen.

`?` in `messagesEndRef.current?.scrollIntoView` — this is optional chaining. If `messagesEndRef.current` is null (component not rendered yet), it won't crash. It just does nothing.

`{ behavior: "smooth" }` — makes the scroll animated instead of instant.

The array `[messages, loading]` is the dependency list — useEffect only runs when these values change. Without this, it would run on every single render.

### sendMessage — The core function

```javascript
const sendMessage = async () => {
  if (!input.trim() || loading) return;
```

**What** — function that runs when you click Send or press Enter.
**`async`** — means this function can do things that take time (like waiting for the API) without freezing the browser.
**`!input.trim()`** — `trim()` removes whitespace. If the input is empty or just spaces, we return early and don't send.
**`|| loading`** — if we're already waiting for a response, don't send again.

---

```javascript
const userMessage = input.trim();
setMessages(prev => [...prev, { role: "user", text: userMessage }]);
setInput("");
setLoading(true);
```

**What** — four things:
1. Save the message text
2. Add it to the messages array immediately — user sees their message right away
3. Clear the input box
4. Set loading to true — shows the typing indicator

We add the message to the screen immediately without waiting for the API. This is called **optimistic UI** — it makes the app feel fast.

---

```javascript
const response = await fetch("http://localhost:8000/chat", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ message: userMessage })
});

const data = await response.json();
setMessages(prev => [...prev, { role: "agent", text: data.response }]);
```

**What** — sends the message to FastAPI and gets the response.
**`fetch`** — JavaScript's built-in HTTP client. Like calling an API.
**`await`** — pauses execution until the fetch completes. Without await, the code would continue before the response arrived.
**`method: "POST"`** — same as your FastAPI `@app.post("/chat")`
**`headers`** — tells FastAPI we're sending JSON
**`JSON.stringify`** — converts JavaScript object `{message: "..."}` into a JSON string
**`response.json()`** — parses the JSON response from FastAPI back into a JavaScript object
**`data.response`** — accesses the `response` field from `{"response": "agent answer here"}`

---

```javascript
} catch (error) {
  setMessages(prev => [...prev, {
    role: "agent",
    text: "Sorry, I'm having trouble connecting..."
  }]);
}
setLoading(false);
```

**What** — error handling and cleanup.
**`try/catch`** — same concept as Python. If `fetch` fails (server is down, network error), the catch block shows an error message instead of crashing.
**`setLoading(false)`** — always runs at the end, whether success or error, to hide the typing indicator.

### handleKeyDown — Enter key support

```javascript
const handleKeyDown = (e) => {
  if (e.key === "Enter" && !e.shiftKey) {
    e.preventDefault();
    sendMessage();
  }
};
```

**What** — listens for key presses in the input field.
**`e`** — the keyboard event object. Contains info about which key was pressed.
**`e.key === "Enter"`** — only trigger on Enter key
**`!e.shiftKey`** — ignore Shift+Enter (that's for new lines in some apps)
**`e.preventDefault()`** — stops the browser's default Enter behaviour (submitting a form)
**`sendMessage()`** — calls our send function

### The JSX — What gets rendered

```javascript
return (
  <div className="app">
    ...
  </div>
);
```

**What** — the HTML-like code that React renders to the screen.
**Why `className` not `class`** — in JSX, `class` is a reserved JavaScript word. React uses `className` instead to avoid conflict.

---

```javascript
{messages.map((msg, index) => (
  <div key={index} className={`message-row ${msg.role === "user" ? "user-row" : "agent-row"}`}>
```

**What** — loops through all messages and renders each one.
**`.map()`** — JavaScript's way of transforming an array. For each message, it returns a JSX element.
**`key={index}`** — React needs a unique key for each item in a list to efficiently track changes.
**Template literal** — the backtick string `` `message-row ${...}` `` lets you embed JavaScript expressions. `msg.role === "user" ? "user-row" : "agent-row"` is a ternary — if user, add "user-row" class, otherwise add "agent-row" class.

---

```javascript
{loading && (
  <div className="message-row agent-row">
    <div className="bubble agent-bubble typing">
      <span></span>
      <span></span>
      <span></span>
    </div>
  </div>
)}
```

**What** — shows the typing indicator when `loading` is true.
**`{loading && (...)}` ** — short-circuit evaluation. If `loading` is false, nothing renders. If `loading` is true, the typing indicator renders. This is React's way of conditional rendering.

The three `<span>` elements are the three bouncing dots — animated purely with CSS.

---

```javascript
<input
  value={input}
  onChange={(e) => setInput(e.target.value)}
  onKeyDown={handleKeyDown}
  disabled={loading}
/>
```

**What** — a controlled input field.
**`value={input}`** — the input displays whatever is in the `input` state variable.
**`onChange`** — every time you type, updates the `input` state with the new value. This makes React the "single source of truth" for what's in the box.
**`disabled={loading}`** — greys out the input while waiting for response.

## App.css — How Styling Works

CSS in React works the same as regular CSS. You target elements by their `className`.

### Key concepts used:

**Flexbox** — used everywhere for layout:
```css
display: flex;
flex-direction: column; /* stack vertically */
align-items: center;    /* center horizontally */
justify-content: space-between; /* space items apart */
```

**`flex: 1`** — on the messages area, this means "take up all available space between the header and input area". The header and input have fixed heights, the messages area fills the rest.

**`flex-direction: row-reverse`** on `.user-row** — this flips the order of avatar and bubble so user messages appear on the right.

**`#E20074`** — Deutsche Telekom's official magenta brand color. Used on the header, user bubbles, and send button.

**The typing animation:**
```css
@keyframes bounce {
  0%, 60%, 100% { transform: translateY(0); }
  30% { transform: translateY(-6px); }
}
```
Defines a keyframe animation called `bounce`. At 30% through the animation, the dot moves up 6px. The three dots have staggered `animation-delay` so they bounce one after another.

## How React, FastAPI, and the Agent connect

```
User types message
        ↓
React state: setMessages adds user message to screen
        ↓
React: fetch POST to http://localhost:8000/chat
        ↓
FastAPI: receives request, calls run_agent()
        ↓
telecom_agent.py: agent loop runs, tools called, answer generated
        ↓
FastAPI: returns {"response": "answer"}
        ↓
React: data.response added to messages state
        ↓
React: re-renders — answer appears on screen
        ↓
useEffect: scrolls to bottom
```

## Key JavaScript concepts used — quick reference

| Concept | What it is | Example |
|---|---|---|
| `const` | Variable that can't be reassigned | `const name = "Dev"` |
| Arrow function | Shorthand function syntax | `(x) => x * 2` |
| `async/await` | Handle asynchronous operations | `const data = await fetch(...)` |
| Spread operator `...` | Copy array/object | `[...prev, newItem]` |
| Optional chaining `?.` | Safe property access | `obj?.property` |
| Ternary operator | Inline if/else | `x > 0 ? "yes" : "no"` |
| Short circuit `&&` | Conditional render | `loading && <Spinner />` |
| Template literal | String with variables | `` `Hello ${name}` `` |
| `.map()` | Transform array | `items.map(item => <div>{item}</div>)` |
| Destructuring | Extract from array/object | `const [a, b] = useState(0)` |
